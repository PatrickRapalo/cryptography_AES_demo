üìö How AES Works - Detailed Explanation ‚ñº

üîê The 4√ó4 State Matrix

AES processes data in 128-bit blocks (16 bytes) arranged in a 4√ó4 grid. Each cell contains one byte.

Example: "Hello World" + padding
H(72) e(101) l(108) l(108) ‚Üí arranged column-by-column into matrix

üîë Secret Key ‚Üí Round Keys

Your password goes through key derivation (PBKDF2) to become a 256-bit key, which then expands into 15 round keys:

  • Round Key 0: Used before Round 1 (initial whitening)
  • Round Keys 1-14: One unique key for each round
  • Why 15? Each layer adds independent security - like 15 locks instead of 1

‚öôÔ∏è The Four Operations

1. SubBytes

Replace each byte using S-box lookup table. Creates confusion - breaks plaintext/ciphertext relationship.

2. ShiftRows

Shift each row left by 0,1,2,3 positions. Creates diffusion - spreads data throughout the matrix.

3. MixColumns

Matrix multiplication within columns. More diffusion - each byte influences all others in its column.

4. AddRoundKey

XOR state with round key. Incorporates your secret - without the key, decryption is impossible.

üîÑ Why 14 Rounds?

More rounds = more security. AES-256 uses 14 rounds (AES-128 uses 10, AES-192 uses 12).

AES-256 Keys: 2256 possibilities
Time to break: Longer than the age of the universe

‚Ü©Ô∏è How Decryption Works

Decryption uses the same key but applies inverse operations in reverse order:

  • Round keys applied in reverse (14 ‚Üí 0)
  • InvSubBytes, InvShiftRows, InvMixColumns - mathematical inverses
  • AddRoundKey is its own inverse (XOR property: A‚äïB‚äïB = A)
Without the correct key: Wrong round keys ‚Üí garbage output
